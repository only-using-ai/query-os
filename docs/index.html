Query Your System Like a Database

A powerful command-line tool that lets you query your filesystem, processes, and applications using familiar SQL syntax. Explore, filter, and analyze your system data with ease.

Terminal
$ q --query "SELECT name, size FROM /home/user WHERE type = 'file' ORDER BY size DESC LIMIT 5"

large_file.zip 2.1 GB
video.mp4 856 MB
backup.tar.gz 234 MB
Query executed in 12.3ms

Installation

Get started with Q in just a few simple steps

Quick Install (Recommended)

# Clone the repository
git clone https://github.com/willdech/query-os.git
cd query-os

# Run the install script
./install.sh

Manual Installation

# Ensure you have Rust installed
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Clone and build
git clone https://github.com/willdech/query-os.git
cd query-os
cargo build --release

# The binary will be in target/release/q
# Optionally install globally
cargo install --path .

Prerequisites

  • • Rust 1.70+ (for manual installation)
  • • macOS, Linux, or Windows

Usage

Learn the basic syntax and start querying your system

Basic Syntax 
q --query "SELECT [fields|*] FROM path [WHERE condition] [ORDER BY field] [LIMIT number]"

Filesystem Fields

  • name - File/directory name
  • type - "file" or "directory"
  • size - File size with units
  • modified_date - Last modification date
  • permissions - Unix permissions
  • path - Relative path
  • extension - File extension

Process Fields

  • pid - Process ID
  • name - Process name/command
  • cpu_usage - CPU usage percentage
  • memory_usage - Memory usage
  • status - Process status

Examples

See Q in action with real-world examples

Filesystem Queries

Find large files

q --query "SELECT name, size FROM . WHERE size > '100 MB' ORDER BY size DESC"

List Rust source files

q --query "SELECT name, path FROM . WHERE extension = 'rs'"

Find recently modified files

q --query "SELECT name, modified_date FROM . ORDER BY modified_date DESC LIMIT 10"

Process Queries

High CPU usage processes

q --query "SELECT pid, name, cpu_usage FROM ps WHERE cpu_usage > '5.0' ORDER BY cpu_usage DESC"

Memory-intensive processes

q --query "SELECT pid, name, memory_usage FROM ps WHERE memory_usage > '500 MB'"

Find specific processes

q --query "SELECT * FROM ps WHERE name LIKE '%chrome%'"

Application Queries

List all applications

q --query "SELECT name, version FROM applications ORDER BY name"

Find developer tools

q --query "SELECT name FROM applications WHERE category LIKE '%developer%'"

Large applications

q --query "SELECT name, size FROM applications WHERE size > '1 GB' ORDER BY size DESC"

Advanced Queries

Subqueries

q --query "SELECT name FROM . WHERE size > (SELECT AVG(size) FROM . WHERE type = 'file')"

Complex conditions

q --query "SELECT * FROM . WHERE extension = 'rs' AND path NOT LIKE '%target%'"

Template usage

q --template large_files --template-args "500 MB"

API Reference

Complete reference for all available fields and query capabilities

Filesystem Fields

name

File or directory name

Type: String

type

"file" or "directory"

Type: String

size

File size with units (B, KB, MB, GB, TB)

Type: String (comparable)

modified_date

Last modification date (YYYY-MM-DD HH:MM:SS)

Type: String (comparable)

permissions

Unix permissions in octal format

Type: String

path

Relative path from query root

Type: String

extension

File extension (lowercase, NULL for directories)

Type: String

Process Fields

pid

Process ID (numeric)

Type: Integer

name

Process name/command

Type: String

cpu_usage

CPU usage percentage (e.g., "5.2%")

Type: String (comparable)

memory_usage

Memory usage with units

Type: String (comparable)

status

Process status (running, sleeping, idle, zombie, stopped)

Type: String

Application Fields

name

Application name

Type: String

version

Version string

Type: String

path

Installation path

Type: String

category

Application category

Type: String

Query Syntax

SELECT [fields|*] FROM source [WHERE condition] [ORDER BY field [ASC|DESC]] [LIMIT number]

WHERE Operators

  • = != > < >= <= - Comparison operators
  • LIKE - Pattern matching (% and _ wildcards)
  • AND OR NOT - Logical operators
  • IN EXISTS - Subquery operators

Data Sources

  • FROM /path/to/dir - Filesystem queries
  • FROM ps - Process queries
  • FROM applications - Application queries
